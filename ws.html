Introduction Active Directory

▼

Les 5 rôles Active Directory

RôleFonction
AD DSAnnuaire — cœur d'AD, gère utilisateurs, ordinateurs, groupes
AD CSPKI — certificats numériques (création, gestion, révocation)
AD FSFédération — SSO entre organisations (relation de confiance)
AD RMSDroits — protection des documents (imprimable, copiable, etc.)
AD LDSVersion légère d'AD — LDAP pour applications

Composants logiques

  • Schéma : définition de chaque objet (classes + attributs)
  • Domaine : regroupement d'objets, limite d'authentification et d'autorisation
  • Arbre : hiérarchie de domaines partageant le nom du parent (ex: fr.ad.net)
  • Forêt : collection d'arbres, schéma commun, initialisée au 1er domaine
  • OU : conteneurs pour organiser, déléguer l'admin et appliquer des GPO

Authentification

  • Autonome (sans AD) : stockée dans SAM, pas de gestion centralisée
  • Domaine (avec AD) : stockage centralisé, relation de confiance
  • NTLM : ancien protocole challenge/response
  • Kerberos : protocole moderne par tickets (TGT → TGS → accès)

Autorisation — SID et ACL

Chaque utilisateur a un SID unique. Lors de l'authentification, un ticket contient son SID + SIDs des groupes. Les ressources sont protégées par des ACLs. Les SIDs du ticket sont comparés aux ACLs.

⚠️ Recréer un utilisateur avec le même nom ne redonne PAS les droits (SID différent).

DNS et AD

AD DS requiert une infrastructure DNS. Le nom de domaine AD DS = nom de domaine DNS. Enregistrements importants : A, AAAA, CNAME, SRV (localisation des DC).

Installation ADDS

▼

Composants Physiques

  • Contrôleur de domaine (DC) : serveur avec le rôle ADDS, contient la base AD, héberge l'authentification
  • RODC (Read Only Domain Controller) : copie en lecture seule, pas de synchro mdp par défaut, pour agences non sécurisées
  • Catalogue Global : copie de tous les objets de la forêt (attributs partiels), améliore la recherche
  • NTDS.dit : base de données AD, par défaut dans %systemRoot%\NTDS
  • SYSVOL : scripts et GPO, par défaut dans %systemRoot%\SYSVOL

Partitions de NTDS.dit

Schéma, Configuration, Domain naming context, Application (ex: DNS)

🎯 TP — Import CSV d'utilisateurs

Import-Module ActiveDirectory
$Users = Import-Csv -Delimiter ";" -Path ".\users.csv"
foreach ($u in $Users) {
  if($u.Firstname -ne "") {
    $Ou = "OU=Users,DC=actus,DC=lan"
    if ($u.OU -eq "Administration") { $Ou="OU=Users,OU=Administration,DC=actus,DC=lan" }
    if ($u.OU -eq "reporters")      { $Ou="OU=Users,OU=Reporters,DC=actus,DC=lan" }
    if ($u.OU -eq "Admins")         { $Ou="OU=Admins,OU=IT,DC=actus,DC=lan" }
    if ($u.OU -eq "WebMaster")      { $Ou="OU=WebMasters,OU=IT,DC=actus,DC=lan" }

    $Password = "P@ssw0rd"
    $FirstLetterFirstname = $u.Firstname.substring(0,1)
    $SamAccount = "$($FirstLetterFirstname).$($u.Lastname)"
    $instance = Get-ADUser "CN=_TemplateAdUsers,OU=Users,OU=Administration,DC=actus,DC=lan" `
      -Properties st,State,StreetAddress,City,co,PostalCode,country

    $Params = @{
      Name              = "$($u.Firstname), $($u.Lastname)"
      SamAccountName    = $SamAccount
      UserPrincipalName = $SamAccount+"@actus.lan"
      DisplayName       = "$($u.Firstname), $($u.Lastname)"
      GivenName         = $u.Firstname
      Surname           = $u.Lastname
      Initials          = $FirstLetterFirstname
      AccountPassword   = (ConvertTo-SecureString $Password -AsPlainText -Force)
      Enabled           = $true
      PasswordNeverExpires = $true
      ChangePasswordAtLogon = $false
      Path              = $OU
      Instance          = $instance
    }
    New-ADUser @Params
  }
}

05

Group Policy Object (GPO)

▼

Concepts clés

  • GPO = ensemble de paramètres pour configurer ordinateurs et utilisateurs (3600+ paramètres)
  • Liaison à : Sites, Domaines, OUs
  • Default Domain Policy : appliquée à tout le domaine — ne toucher que pour la sécurité générale
  • Default Domain Controllers Policy : spécifique aux DC

⚡ Héritage GPO — Ordre de priorité

Du moins prioritaire au plus prioritaire :

1. Stratégie locale → 2. Site → 3. Domaine → 4. OU → 5. OU enfant

  • Bloquer l'héritage : les GPOs supérieures ne s'appliquent plus aux enfants
  • Forcer (Enforced) : s'applique même si l'héritage est bloqué
  • En cas de conflit : dernière appliquée gagne (OU enfant > OU parent)

Rafraîchissement

  • Au démarrage de l'ordinateur / ouverture de session
  • Par défaut toutes les 90 minutes
  • Forcer : GPUpdate /force

🔧 Résultat GPO

# Rapport HTML complet
gpresult /H c:\Report.html

# Rapport rapide dans le terminal
gpresult /R

Filtrage

  • WMI Filtering : filtrer par hardware/OS. Namespace root\CIMv2
  • Security Filtering : spécifier quels utilisateurs/ordinateurs reçoivent la GPO
# Exemple WMI : seulement Windows 10
SELECT * FROM Win32_OperatingSystem WHERE Version LIKE "10.0%" AND ProductType = "1"

# ProductType: 1=client, 2=DC, 3=serveur non-DC
# Type 6.2/6.3 = Windows 8/8.1, 10.0 = Windows 10

Déploiement d'applications MSI

  • Attribué (Assigned) : installé automatiquement au démarrage/ouverture de session
  • Publié (Published) : l'utilisateur peut l'installer lui-même via Panneau de config
  • Utiliser un chemin UNC : \\Server\Share\app.msi
  • Connexion lente ≤ 500 kbit/s : certaines GPO pas appliquées

Profils Itinérants (Roaming Profiles)

  • Redirection de dossiers : dossiers perso déplacés sur un serveur de partage
  • Profil itinérant : copie le profil depuis le serveur à l'ouverture de session, met à jour à la fermeture
  • Processus : créer groupe sécurité → créer dossier partagé → créer GPO → configurer

Loopback Processing

  • Merge : paramètres ordinateur + utilisateur fusionnés (ordinateur prioritaire si conflit)
  • Replace : seuls les paramètres ordinateur s'appliquent
Computer Configuration > Admin Templates > System > Group Policy > Configurer le mode de traitement par bouclage

Stockage

  • GPC (Group Policy Container) : stocké dans AD, identifié par GUID
  • GPT (Group Policy Template) : fichiers dans SYSVOL
  • Central Store : \\domaine\SysVol\domaine\Policies\PolicyDefinitions
  • ADMX (paramètres XML) + ADML (langues) remplacent les anciens ADM

🔧 Backup/Restore GPO

Backup-GPO -All -Path "C:\GPOBackup"
Restore-GPO -Name "NomGPO" -Path "C:\GPOBackup"

Scripts GPO

Placer dans C:\Windows\SYSVOL\domain\Scripts. Chemin UNC : \\Server\NETLOGON

Options : ouverture/fermeture de session. Langages : PowerShell, VBScript, .BAT, .CMD

Réplication

  • Seules les modifications sont répliquées (pas l'objet entier, sauf à la création)
  • KCC (Knowledge Consistency Checker) : crée auto les liens de réplication
  • Topologie par défaut : ≥ 2 chemins, ≤ 3 sauts entre DC
  • Réplication intersite : 180 min par défaut
  • Protocoles : DS-RPC (par défaut, IP) ou ISM-SMTP (sites déconnectés)
  • Bridgehead : serveur principal de réplication d'un site

Administration Avancée

▼

Password Settings Objects (PSO)

  • Politiques de mdp affinées — s'appliquent à des utilisateurs ou groupes (pas des OUs !)
  • Priorité configurable (Precedence) : plus bas = plus prioritaire
  • Si un user a un PSO direct + un PSO via groupe → le PSO direct gagne
  • Vérifier : User > Propriété > Éditeur d'attribut > msDS-ResultantPSO

🎯 TP — Créer un PSO pour Reporters (PowerShell)

$templatePSO = New-Object Microsoft.ActiveDirectory.Management.ADFineGrainedPasswordPolicy

$templatePSO.DisplayName = "ReporterPso"
$templatePSO.Precedence = 1
$templatePSO.ReversibleEncryptionEnabled = $false
$templatePSO.PasswordHistoryCount = 5
$templatePSO.ComplexityEnabled = $false
$templatePSO.MinPasswordLength = 8
$templatePSO.MinPasswordAge = [TimeSpan]::Parse("0.01:00:00")
$templatePSO.MaxPasswordAge = [TimeSpan]::Parse("30.00:00:00")
$templatePSO.LockoutThreshold = 3
$templatePSO.LockoutObservationWindow = [TimeSpan]::Parse("0.01:00:00")
$templatePSO.LockoutDuration = -9223372036854775808   # = jamais (admin déverrouille)

New-ADFineGrainedPasswordPolicy -Instance $templatePSO -Name "ReporterPso" `
  -OtherAttributes @{'msDS-PSOAppliesTo'='CN=GpPsoReporters,OU=Groups,OU=Reporters,DC=actus,DC=lan'}

-9223372036854775808 = valeur pour spécifier "jamais" (admin doit déverrouiller manuellement)

Sécurité & Pare-feu Windows

▼

Fondamentaux de la sécurité

  • Moindre privilège : comptes séparés par rôle (Admin SQL, Admin AD, compte standard)
  • PowerShell JEA (Just Enough Administration) : limite les actions + trace tout
  • Par défaut sur Windows Server : pare-feu actif, découverte réseau désactivée, UAC actif

Windows Filtering Platform (WFP)

Architecture de filtrage réseau. Couche 2+, filtre paquets TCP/IP, IPsec, RPC. Composants :

  • Base Filtering Engine (BFE) : cœur de WFP, service bfe.dll dans svchost.exe
  • Filter Engine : ~50 couches noyau + ~10 couches utilisateur
  • Callouts : fonctionnalités étendues (antivirus, IDS, NAT, contrôle parental)
  • Shims : entre pile réseau et moteur de filtre, extraction/classification
  • Layers & Sublayers : organisent les filtres. Les filtres ont un poids (décroissant).

Pare-feu Windows — Profils

ProfilDescription
DomaineAuto quand joint à un AD — ne peut pas être défini manuellement
PrivéRéseaux domestiques — peut être défini manuellement
PublicPar défaut pour réseaux non identifiés — sécurité max

Comportement par défaut : bloquer entrant (sauf sollicité/règle) + autoriser sortant (sauf règle)

Priorité des règles

  1. Les règles d'autorisation explicites > blocage par défaut
  2. Les règles de blocage explicites > toute règle d'autorisation en conflit
  3. Les règles plus spécifiques > règles moins spécifiques (sauf blocage explicite)

⚠️ Le pare-feu Windows ne supporte pas l'ordre/pondération des règles.

🔧 PowerShell Pare-feu

# Voir le profil réseau actif
Get-NetConnectionProfile

# Activer/désactiver le pare-feu
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False

# Créer une règle (ouvrir port 443)
New-NetFirewallRule -DisplayName "Allow Web 443" -Direction Inbound `
  -Protocol TCP -LocalPort 443 -Action Allow

# Modifier une règle (restreindre à une IP)
Set-NetFirewallRule -DisplayName "Allow Web 443" -RemoteAddress 172.16.20.100

# Supprimer une règle
Remove-NetFirewallRule -DisplayName "Allow Web 443"

🔧 CMD Pare-feu (netsh)

# Voir tous les profils
netsh advfirewall show all

# Activer/désactiver
netsh advfirewall set allprofiles state on
netsh advfirewall set allprofiles state off

# Créer une règle
netsh advfirewall firewall add rule name="Allow Web 443" dir=in protocol=TCP localport=443 action=allow

# Modifier une règle
netsh advfirewall firewall set rule name="Allow Web 443" new remoteip=172.16.20.100

# Supprimer une règle
netsh advfirewall firewall delete rule name="Allow Web 443"

Best Practices pare-feu en domaine

  • Gérer de manière centralisée via GPO
  • Créer une GPO de base pour tous + GPOs spécifiques par rôle
  • Activer les 3 profils, activer la journalisation
  • Désactiver la fusion des règles locales si nécessaire
  • Convention de nommage des règles

Outils de sécurité

  • Microsoft SCT (Security Compliance Toolkit) : modèles de sécurité + PolicyAnalyzer + LGPO
  • PingCastle : audit AD gratuit, score de risque, rapport détaillé
  • Microsoft LAPS : mdp admin local unique par machine, stocké dans AD, rotation auto

⚠️ LAPS — Attention 2 versions

  • Legacy LAPS : ancienne, Win Server 2003-2022
  • Windows LAPS : nouvelle (après mise à jour 11/04/2023), chiffrement AD, historique mdp, Entra ID
