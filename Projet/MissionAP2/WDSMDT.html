Contexte :

Dans ce projet, nous avons exploré les solutions de déploiement multicast et de clonage de postes de travail. Nous avons comparé plusieurs outils, dont Windows Deployment Services (WDS) et Microsoft Deployment Toolkit (MDT), et avons mis en place la solution retenue pour les tests et le tutoriel.

Q'est ce que sont WDS et MDT?

WDS (Windows Deployment Services)

WDS est une solution de déploiement d'images Windows. Elle permet de déployer des systèmes d'exploitation sur plusieurs machines à travers un réseau en utilisant un serveur central. WDS est bien adapté pour les environnements Windows mais peut nécessiter des outils supplémentaires pour la gestion avancée des images.

MDT (Microsoft Deployment Toolkit)

Microsoft Deployment Toolkit (MDT) permet de personnaliser les images système (ajout d'applications, configurations spécifiques, etc.) avant de les importer dans Windows Deployment Services (WDS). Une fois l'image configurée et importée, WDS s'occupe de son déploiement sur les postes via des méthodes comme le multicast.

Mise en Place de la Solution Retenue

Après une analyse approfondie, nous avons opté pour MDT comme solution de déploiement et de clonage des postes de travail. Voici les étapes de la mise en place :

  1. Installation de WDS et MDT
  2. Configuration de WDS et MDT
  3. Ajout et configuration d'un systeme d'exploitation avec MDT
  4. Création d'une tâche de déploiement avec MDT
  5. Test du déploiement multicast
  6. Ajout des logiciels
  7. Création d'un tutoriel

Rédaction du Tutoriel

Un tutoriel détaillé a été rédigé pour guider les utilisateurs dans l'utilisation de MDT pour le déploiement et le clonage de postes de travail. Il comprend des instructions étape par étape, des captures d'écran et des conseils pour éviter les erreurs courantes.
